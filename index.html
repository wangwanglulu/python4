4.1 Dictionaries (1)

  • A dictionary is like a list, but more general. In a list, the index positions have to be integers; in a dictionary, the indices can be (almost) any type.


>>> eng2sp = dict() #This is False
>>> print(eng2sp)
{}

  • You can think of a dictionary as a mapping between a set of indices (which are called keys) and a set of values.


>>> eng2sp['one'] = 'uno'
>>> print(eng2sp)
{'one': 'uno'}

  • In general, the order of items in a dictionary is unpredictable.


>>> eng2sp = {'one': 'uno', 'two': 'dos', 'three': 'tres'}
>>> print(eng2sp)
{'one': 'uno', 'three': 'tres', 'two': 'dos'}

>>> print(eng2sp['two'])
'dos'

  • The len function works on dictionaries; it returns the number of key-value pairs:

    • 
>>> eng2sp = {'one': 'uno', 'two': 'dos', 'three': 'tres'}
>>> len(eng2sp)
3

  • The in operator works on dictionaries. (keys)

    • 
>>> eng2sp = {'one': 'uno', 'two': 'dos', 'three': 'tres'}
>>> 'one' in eng2sp
True
>>> 'uno' in eng2sp
False

4.2 Dictionaries (2)

Lists vs. Dictionaries

  • Lists: in operator, insert(), del (slow)

  • Dictionaries: in operator, add key-value pair, del (fast)

  • Hash Function

  • The key should be immutable. (A list cannot be a key)

  • To see whether something appears as a value in a dictionary, you can use the method values, which returns the values as a list, and then use the in operator:


>>> eng2sp = {'one': 'uno', 'two': 'dos', 'three': 'tres'}
>>> vals = list(eng2sp.values())
>>> 'uno' in vals
True

  • Dictionary as a set of counters

  • Suppose you are given a string and you want to count how many times each letter appears. 


word = 'brontosaurus'
d = dict()
for c in word:
    if c not in d:
        d[c] = 1
    else:
        d[c] = d[c] + 1
print(d)

{'b': 1, 'r': 2, 'o': 2, 'n': 1, 't': 1, 's': 2, 'a': 1, 'u': 2}

  • Dictionaries have a method called get that takes a key and a default value.


>>> counts = { 'chuck' : 1 , 'annie' : 42, 'jan': 100}
>>> print(counts.get('jan', 0)) #0 is the default value
100
>>> print(counts.get('anni', 1550))
1550

  • We can use get to write our histogram loop more concisely.


word = 'brontosaurus'
d = dict()
for c in word:
    d[c] = d.get(c,0) + 1
print(d)

Nesting

  • You can nest a set of dictionaries inside a list, a list of items inside a dictionary, or even a dictionary inside another dictionary.

  • A List of Dictionaries


alien_0 = {'color': 'green', 'points': 5}

alien_2 = {'color': 'red', 'points': 15}

aliens = [alien_0, alien_1, alien_2]

for alien in aliens:
    print(alien)

{'color': 'green', 'points': 5}

{'color': 'red', 'points': 15}

Dictionaries: Summary

  • Elements are key-value pairs. Empty dictionary {}

  • Features：unpredictable order, key-value pairs are mutable (keys are immutable, but you can modify values). Do not make repeated keys.

  • Looping: .items(), .keys(), .values()

    .get() (value or default value)

4.3 Tuples

  • A tuple is a sequence of values much like a list.
  • The important difference is that tuples are immutable.

>>> t = ('a', 'b', 'c', 'd', 'e')


>>> t = tuple() #Empty is False
>>> print(t)
()

  • To create a tuple with a single element, you have to include the final comma:


>>> t1 = ('a',)
>>> type(t1)
tuple

>>> t2 = ('a')
>>> type(t2)
str

  • Most list operators also work on tuples. The bracket operator indexes an element:


>>> t = ('a', 'b', 'c', 'd', 'e')
>>> print(t[0])
'a'

>>> print(t[1:3])
('b', 'c')

>>> t[0] = 'A'
TypeError: object doesn't support item assignment

  • You can’t modify the elements of a tuple, but you can replace one tuple with another: 


>>> t = ('A',) + t[1:]
>>> print(t)
('A', 'b', 'c', 'd', 'e')

Look at this!


>>> t = ('a', 'b', ['A', 'B'])
>>> t[2][0] = 'X'
>>> t[2][1] = 'Y'
>>> t
('a', 'b', ['X', 'Y'])


>>> t = ('a', 'b', ('A', 'B'))
>>> t[2][0] = 'X'
Traceback (most recent call last):
  File "test.py", line 2, in 
    t[2][0] = 'X'
TypeError: 'tuple' object does not support item assignment

  • You cannot modify the elements of a tuple, but you can replace one tuple with another.
